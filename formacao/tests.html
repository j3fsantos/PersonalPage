QUnit Testing

test(nameOfTheTest, testFunction):

  • nameOfTheTest: string with the name of the test

  • testFunction: function with the test itself

  • Calling test() just adds the test to a queue, and its execution is deferred and controlled by the test runner


Testing options:

  • Check for Globals: Make sure your test does not add variables to the global object

  • No try-catch: If the content of the test throws an exception, the test will die

  • Below the header, there is a summary, showing the total time it took to run all tests as well as the overall number of total and failed assertions

Testing Assertions

Essential elements of any unit test are assertions

  • ok(arg [, message]): the assertion passes if the argument evaluates to true

  • equal(arg1,arg2[, message]): the assertion passes if arg1 is equal to arg2 using the == comparison

  • deepEqual(arg1,arg2[, message]): it uses === instead of == and it compares the content of the objects
